## 2) Flujo de pedidos (Supabase)

Tu proyecto trabaja con estas 3 tablas:

- productos
- pedidos_items
- sugerencias

No se usa tabla pedidos para guardar compras.

### Como se guarda un pedido

- Cada compra genera un pedido_ref unico (ej: 0001).
- En pedidos_items se guarda 1 fila por producto del carrito.
- Todas las filas del mismo pedido comparten el mismo pedido_ref.
- En el panel admin, el sistema agrupa por pedido_ref para mostrar 1 pedido completo.

Esto cumple con: un ID de pedido para toda la compra, aunque existan varias lineas de productos.

## 3) Supabase SQL recomendado (minimo)

```sql
create table if not exists public.pedidos_items (
  id bigint generated by default as identity primary key,
  pedido_ref text not null,
  producto_id bigint,
  producto_nombre text not null,
  categoria text,
  precio integer not null default 0,
  qty integer not null default 1,
  created_at timestamptz not null default now()
);

create index if not exists idx_pedidos_items_ref on public.pedidos_items (pedido_ref);
```

## 4) Policies RLS sugeridas (anon)

Si usas clave publica anon desde frontend, necesitas policies para leer/escribir.

```sql
alter table public.pedidos_items enable row level security;

create policy if not exists "pedidos_items_select_anon"
on public.pedidos_items
for select
to anon
using (true);

create policy if not exists "pedidos_items_insert_anon"
on public.pedidos_items
for insert
to anon
with check (true);

create policy if not exists "pedidos_items_delete_anon"
on public.pedidos_items
for delete
to anon
using (true);
```

Para sugerencias aplica el mismo patron en public.sugerencias (select/insert segun tu necesidad).

## 5) Git privado + Netlify

## 2) Flujo de pedidos (Supabase)

Tu proyecto trabaja con estas 3 tablas:

- productos
- pedidos
- sugerencias

Toda la compra se guarda en pedidos (1 fila por pedido_ref).

### Como se guarda un pedido

- Cada compra genera un pedido_ref unico (ej: 0001).
- Se guarda una sola fila en pedidos.
- El detalle de productos va en items (jsonb), con nombre/precio/qty/categoria.
- En el panel admin se lee directo desde pedidos, sin agrupacion adicional.

## 3) Supabase SQL recomendado (minimo)

```sql
create table if not exists public.pedidos (
  id bigint generated by default as identity primary key,
  pedido_ref text not null unique,
  cliente_nombre text,
  cliente_telefono text,
  direccion text,
  depto text,
  comuna text,
  referencia text,
  notas text,
  comprobante_declarado boolean default false,
  subtotal integer not null default 0,
  delivery_fee integer not null default 0,
  total integer not null default 0,
  estado text not null default 'pendiente',
  items jsonb not null default '[]'::jsonb,
  status_historial jsonb not null default '[]'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

## 4) Policies RLS sugeridas (anon)

Si usas clave publica anon desde frontend, necesitas policies para leer/escribir.

```sql
alter table public.pedidos enable row level security;

create policy if not exists "pedidos_select_anon"
on public.pedidos
for select
to anon
using (true);

create policy if not exists "pedidos_insert_anon"
on public.pedidos
for insert
to anon
with check (true);

create policy if not exists "pedidos_update_anon"
on public.pedidos
for update
to anon
using (true)
with check (true);

create policy if not exists "pedidos_delete_anon"
on public.pedidos
for delete
to anon
using (true);
```

Para sugerencias aplica el mismo patron en public.sugerencias (select/insert segun tu necesidad).

## 5) Git privado + Netlify

Si quieres repositorio privado y deploy estable:

1. En GitHub, deja el repo en Private.
2. En Netlify, conecta GitHub y autoriza acceso al repo privado.
3. Build command: vacio.
4. Publish directory: / (raiz del repo).
5. Deploy branch: main.

Este proyecto es estatico, no necesita build.
